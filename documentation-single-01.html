Setup a reverse-proxy with Cloudflare

Go to “Workers”

Choose “Manage Workers

Click “Create a Worker”

On the left “script” side, replace the code with the following one (replace 'www.client.com' and client.backend.verbolia.com values with your hostname and the given verbolia hostname) and save:

async function handleRequest(request) { let url = new URL(request.url) // Check if incoming hostname is // a key in the ORIGINS object let target = ORIGINS[url.hostname] // If it is, proxy request to that third party origin if (target) { url.hostname = target return fetch(url, request) } // Otherwise, process request as normal return fetch(request) } addEventListener('fetch', event => { event.respondWith(handleRequest(event.request)) }) /** * An object with different url's to fetch * @param {Object} ORIGINS */ const ORIGINS = { 'www.client.com': 'client.backend.verbolia.com', }

Rename the worker to “reverse-proxy-verbolia:

Go back to workers main page and click “add route”:

Fill the Route information (replace “www.client.com” with your hostname and “path-to-verbolia” with the agreed routed url pattern) and choose the previously defined “reverse-proxy-verbolia” as Worker
